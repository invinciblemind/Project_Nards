Руководство разработчика
========================

Подготовка окружения
--------------------

1. Установите Python 3.12.
2. Создайте виртуальное окружение.
3. Установите проект в editable-режиме с dev-зависимостями.

.. code-block:: bash

   python -m venv .venv
   .venv\Scripts\activate
   python -m pip install --upgrade pip
   python -m pip install -e .[dev]

Основные команды
----------------

.. code-block:: bash

   python -m doit lint
   python -m doit test
   python -m doit coverage
   python -m doit docs
   python -m doit build
   python -m doit check

Запуск сетевой игры (LAN)
--------------------------

Для игры по локальной сети на разных компьютерах:

1. **Хост** (тот, кто создаёт игру) запускает:

   .. code-block:: bash

      nardy --server --socket-host 0.0.0.0 --socket-port 8765

   Или через меню: ``LAN Game -> Host Game``.

2. **Клиент** подключается к IP-адресу хоста (например, 192.168.1.10):

   .. code-block:: bash

      nardy --join --socket-host 192.168.1.10 --socket-port 8765

   Или через меню: ``LAN Game -> Join Game`` и ввод IP/порта.

Примечания:
- Убедитесь, что брандмауэр разрешает входящие подключения на указанном порту.
- Сервер использует авторитарную модель: все ходы валидируются на хосте.
- При разрыве соединения игра завершается с сообщением об ошибке.

Соглашения по коду
------------------

* Публичные классы и функции должны иметь типы и docstring.
* Доменный слой должен оставаться изолированным от tkinter и socket API.
* Новая функциональность добавляется небольшими PR с тестами и документацией.
